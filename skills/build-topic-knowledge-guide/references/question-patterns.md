# Question Patterns

Use these patterns to create the guide's Q&A spine. Prefer specific questions over generic headings.

## Mechanism Questions

Use when the learner must explain how something works.

Question forms:

- What happens step by step when [event]?
- How does [component] turn [input] into [output]?
- Why does [condition] cause [result]?
- What is the lifecycle of [object/process]?

Answer shape:

1. Direct answer.
2. Ordered steps with inputs, actors, state changes, and outputs.
3. Example.
4. Verification method.
5. Key concepts.

## Difference Questions

Use when learners confuse similar terms.

Question forms:

- What is the difference between [A] and [B]?
- [A] vs [B] - when does each matter?
- Why is [A] not the same as [B]?

Answer shape:

| Concept | Owns/does | Best for | Watch out |
|---|---|---|---|

Then add a concrete scenario and a default decision rule.

## Procedure Questions

Use when the learner must perform a task.

Question forms:

- How do you [perform task]?
- What is the preferred way to [achieve outcome]?
- What commands, steps, or artifacts do you use for [task]?

Answer shape:

1. Preferred path.
2. Inputs and setup.
3. Commands or steps.
4. Expected output.
5. Alternatives.
6. Safety/cost/state cautions.

## Troubleshooting Questions

Use when the learner must diagnose failures.

Question forms:

- [System] is slow/broken/wrong. How do you walk through it?
- What causes [symptom], and how do you prove it?
- How do you distinguish [failure A] from [failure B]?

Answer shape:

1. Stabilize or observe.
2. Narrow the scope.
3. Test the highest-probability causes.
4. Fix the confirmed cause.
5. Verify recovery.
6. Prevent recurrence.

Include a symptom table.

## Compare-and-Choose Questions

Use when multiple approaches are credible.

Question forms:

- [A] vs [B] - when should you use which?
- What trade-offs decide between [options]?
- What is the default recommendation and when would you override it?

Answer shape:

| Option | Choose when | Avoid when | Trade-off |
|---|---|---|---|

Then state:

- Default stance.
- Caveat.
- Evidence that would change the recommendation.

## Boundary Questions

Use when learners overgeneralize.

Question forms:

- What does [concept/tool] not guarantee?
- Where does [approach] fail?
- What assumptions must be true?

Answer shape:

- Guarantees.
- Non-guarantees.
- Required assumptions.
- Counterexample.
- Safer alternative.

## Recall Questions

Use for final revision.

Question forms:

- Explain [mechanism] from memory.
- Draw [flow] and label each transition.
- List the first checks for [symptom].
- Give the decision rule for [A] vs [B].

Answer shape:

- Expected answer.
- Common missing points.
- Self-score criteria.
